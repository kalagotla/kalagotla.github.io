Overview

The Lagrangian Particle Tracking Library (lptlib) is a highly-parallelized, open-source Python library designed for accurate particle tracking in supersonic and compressible flows. Unlike existing post-processing codes, lptlib provides comprehensive features required to study uncertainty in tracer particle response, making it ideal for particle image velocimetry (PIV) uncertainty quantification and particle dynamics history (PDH) analysis.

The library addresses critical needs in computational fluid dynamics by offering easy access to source code for algorithm customization, seamless integration of multiple drag models, support for polydisperse particle simulations, and robust data compatibility for post-processing workflows. Built with modularity and performance in mind, lptlib enables researchers to track particles through complex flow fields while quantifying the effects of particle inertia, compressibility, and rarefaction.

Key Features

Core Capabilities

  • Three Core Aspects: Search, Interpolation, and Integration algorithms with multiple implementation options
  • Multiple Drag Models: Stokes, Schiller-Nauman, Melling, Tedeschi, Loth, and custom drag formulations
  • High-Performance Parallelization: MPI-based parallel computing for large-scale simulations
  • Frame Transformation: Convert Lagrangian particle data to Eulerian fields for visualization and analysis
  • Polydisperse Simulation: Support for particle size distributions to quantify uncertainty
  • Modular Design: Easy-to-understand code structure with minimal background required
  • Data Compatibility: Seamless integration with Plot3D format and other CFD data formats

Algorithm Overview

The Lagrangian Particle Tracking algorithm consists of three fundamental aspects that work together to accurately trace particle paths through flow fields:

Lagrangian Particle Tracking Algorithm Workflow

1. Search

The search aspect locates particles within structured curvilinear grids. Two primary approaches are implemented:

  • Physical Space (p-space): Uses Newton-Raphson iteration to find particle locations in physical coordinates, providing high accuracy for complex geometries
  • Computational Space (c-space): Leverages orthonormal computational grids for O(1) search complexity, ideal for large-scale simulations

2. Interpolation

Once a particle's location is found, flow field data must be interpolated to that point. Multiple interpolation schemes are available:

  • Trilinear Interpolation: Standard second-order accurate method with C⁰ continuity
  • Radial Basis Functions (RBF): Higher-order interpolation for scattered data with customizable shape parameters
  • Adaptive Schemes: Automatically switches to nearest-neighbor interpolation in shock cells for improved accuracy

3. Integration

Particle trajectories are computed using numerical integration of the quasi-steady drag equation:

  • Runge-Kutta 2nd Order (RK2): Fast integration scheme suitable for smooth flows
  • Runge-Kutta 4th Order (RK4): Higher accuracy for complex flow features and shock interactions
  • Adaptive Time-Stepping: Automatically adjusts time steps based on particle acceleration and Courant number constraints

Workflow: Lagrangian to Eulerian Transformation

One of the key capabilities of lptlib is transforming scattered Lagrangian particle data into structured Eulerian fields. This enables visualization and further analysis of particle dynamics history (PDH) effects in flow measurements.

Workflow: CFD data → Particle paths → Eulerian field transformation

The transformation process uses Delaunay triangulation to map scattered particle data onto regular grids, ensuring smooth transitions even in high-gradient regions like shock interfaces. This capability is essential for generating PDH-informed synthetic PIV data and understanding how particle inertia affects measured velocity fields.

Installation & Usage

Installation

Install lptlib directly from PyPI: 

pip install lptlib

Quick Start

Here's a simple example of tracking particles through a flow field: 

from lptlib import Streamlines

# Initialize the streamline tracker
streamline = Streamlines(
    grid_file='grid.x',
    flow_file='flow.q',
    point=[0.1, 0.1, 0.05],  # Starting point
    method='pRK4',  # Runge-Kutta 4th order in physical space
    diameter=1e-6,  # Particle diameter (meters)
    density=1000,   # Particle density (kg/m³)
    time_step=1e-3  # Initial time step
)

# Compute particle path
streamline.compute(method='adaptive-ppath')

Features in Detail

  • Multiple Tracking Methods: p-space, c-space, adaptive variants, and unsteady tracking
  • Drag Models: Choose from Stokes, Schiller-Nauman, Melling, Tedeschi, Loth, or implement custom models
  • Parallel Processing: Built-in MPI support for tracking multiple particles simultaneously
  • Data Export: Export particle paths in NumPy format for further analysis
  • Frame Conversion: Transform Lagrangian data to Eulerian fields using the DataIO class
